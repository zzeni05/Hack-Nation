# Demo Video Script

Record Sunday morning, 7-9 AM. Target: 90-120 seconds. Tool: Loom or ScreenStudio.

## Structure (time budget)

**Hook (10s)** — one sentence, problem + who has it.
> "[Target user] spends [X hours / loses $Y / can't do Z] because [problem]."

**Demo (60s)** — screen recording of happy path, no narration fluff.
Voiceover hits:
- What you type / input
- What the system does
- The output + the metric (show the number on screen)

**How it works (15s)** — one diagram, one sentence per layer.
> "Input hits [X], which calls [Y], evaluated against [Z], returns [W]."

**Metric + ask (15s)** — the quantifiable thing + what's next.
> "In testing, it [hits X% / runs in Yms / processes Z docs]. Next is [concrete step]."

## Recording checklist

- [ ] Close every other tab. Hide dock, notifications, Slack.
- [ ] Demo environment uses a clean account / fresh data. No personal stuff visible.
- [ ] Deploy is live and tested 30 min before recording. Test on incognito.
- [ ] Audio: quiet room, one take, script in front of you. Don't improvise.
- [ ] Export 1080p. Upload to YouTube unlisted OR Loom. Put link in README.

## Don't

- Don't open with "Hi, I'm Zen and today I'm going to show you..."
- Don't explain the tech stack until the "how it works" beat
- Don't apologize for anything ("sorry this is rough", "still early" — cut)
- Don't run past 2 minutes. Judges have 200 submissions.
